نمای کلی

رد جی اس یک فریم ورک سبک جاوا اسکریپتی ، به جهت رابطه آسان تر Front و Back می باشد ؛ این فریم ورک در حال حاضر به نسخه 1.0 خودش رسیده و همراه با رد فریم ورک ارائه شده است .

از قابلیت های رد جی اس میتوان به موارد زیر اشاره کرد :


- پیروی از الگوی طراحی MVC

- کد نویسی درون تگ های HTML

- دسترسی آسان به المان و تغییر ویژگی آن

- ارسال درخواست AJAX

- مدیریت کوکی ها

- افکت گرافیکی

- سرویس ارزیابی ، فیلتر و زدودن رشته

رخداد آماده بودن سند

آماده بودن سند ، یک رخداد از پیش تعیین شده در HTML است ؛ زمانی که سند به طور کامل دانلود و رندر شده این رخداد فراخوانده می شود .

فرض کنید برای المانی که هنوز لود نشده Event Listener ثبت کنیم ؛ اگر از رخداد آماده بودن سند استفاده نکنیم خطا رخ می دهد .
از آنجا که استفاده از این رخداد بسیار رایج و مورد استفاده بوده رد جی اس تابع ای را به آن اختصاص داده است .

نحوه استفاده از تابع ready که آرگومان ورودی آن یک Callback است :
Red.ready(function () {
    Root.initialize();
});

ماژول

در رد جی اس ، ماژول اشاره به فضای نام برنامه دارد و هدف آن جلوگیری از اشتباه گرفتن کنترلر های همنام است .
یک سند می تواند دارای چندین ماژول و هر ماژول دارای چندین کنترلر باشد .

اضافه کردن ماژول
<body>
    <div red-app="main"></div>
    <script src="Dependencies/Library/Red.min.js"></script>
        <script>
            Red.app.module("main");
            Red.ready(function () {
                // some code
            });
        </script>
</body>

توجه داشته باشید که رد جی اس به کمک ویژگی المان شما را قادر می سازد تا درون HTML از جاوا اسکریپت استفاده کنید ؛
در قطعه کد بالا ، المان div رو با ویژگی red-app به عنوان ماژول main انتخاب کردیم و سپس به کمک تابع module آن را در حافظه ثبت کرده ایم .

کنترلر

کنترلر در طراحی رد جی اس نقش دریافت داده از مدل (کلاینت) ، تصمیم گیری بر اساس شرایط ، ثبت داده در حافظه در صورت لزوم و ارسال داده به لایه View رو بر عهده دارد .

افزودن کنترلر
<body>
    <div red-app="main">
        <div red-controller="test"></div>
    </div>

    <script src="Dependencies/Library/Red.min.js"></script>
        <script>
            Red.app.module("main");
            Red.app.addController("main", "test", function () {
                // decisions
            });

            Red.ready(function () {
                // some code
            });
        </script>
</body>

در مثال بالا ابتدا ماژول main رو ثبت کردیم ؛ سپس کنترلمون رو به ماژول main با تابع addController اضافه کردیم .

توجه داشته باشید که پارامتر سوم تابع addController از نوع Callback می باشد و تمامی متغیر های استفاده شده در این Callback در حافظه ثبت می شوند .


تمامی این متغیر های ثبت شده در حافظه قابل استفاده در تگ های HTML می باشند .

جای گذاری داده (Data Binding)
<body>
    <div red-app="main">
        <div red-controller="test">
          <div id="result" red-bind="$calculation_result"></div>
        </div>
    </div>

    <script src="Dependencies/Library/Red.min.js"></script>
        <script>
            Red.app.module("main");
            Red.app.addController("main", "test", function () {
                $calculation_result = 3243;
            });

            Red.ready(function () {
                // some code
            });
        </script>
</body>

نتیجه در المان result id ظاهر خواهد شد .


یک کنترلر میتواند علاوه بر متغیر هایی که در Callback ثبت می شوند ، مقادیر اولیه هم داشته باشد :

مقادیر اولیه کنترلر
<body>
    <div red-app="main" red-controller="test" red-init="var user_id = 43;">
        <div id="user-id" red-bind="user_id"></div>
    </div>
    <script src="Dependencies/Library/Red.min.js"></script>
        <script>
            Red.app.module("main");
            Red.app.addController("main", "test", function () {
                // decisions
            });

            Red.ready(function () {
                // some code
            });
        </script>
</body>

در مثال بالا کنترلر test رو به ماژول main اضافه کردیم ؛ سپس در مقدار دهی اولیه ، متغیر 'user_id' رو مقدار دهی کرده

و در آخر متغیر 'user_id' رو در المان 'user-id' جای گذاری کردیم .

داده بازگشتی به ویو :

ابتدا یک ماژول ساخته و کنترلر مربوطه رو بهش اضافه کردیم .

در مرحله آخر یک المان Feeder ساختیم با ویژگی red-bind به محتوای ثبت شده در حافظه دسترسی گرفیتم .

مدیریت مسیر

در رد جی اس ، سیستمی برای مدیریت آسان تر مسیر ها و لینک ها در اچ تی ام ال ارائه شده است .

مدیریت مسیر ها با رد جی اس
<div red-app="main">
    <a href="#!blog">Blog</a>
    <a href="#!forum">Forum</a>
</div>

<script>
    var blog_path = "http://redframework.ir/blog";
    var forum_path = "http://redframework.ir/forums";
    var module_name = "main";
    Red.app.registerRoute(module_name, {
        blog: blog_path,
        forum: forum_path,
    });
</script>

توضیح :

به کمک تابع registerRoute از ماژول اپلیکیشن ، 2 مسیر blog_path و forum_path را ثبت کرده ایم .

این تابع دو پارامتر ضروری دریافت می کند -> اسم ماژول و ابجکتی حاوی نام مسیر به عنوان کلید و آدرس مسیر به عنوان مقدار .

* مسیر ها فقط در ماژول مربوطه خودشان قابل دسترسی هستند .
* دسترسی به مسیر ها درون اچ تی ام ال با کاراکتر های ویژه (#!نام_مسیر) امکان پذیر می باشد .

دستور GET و SELECT

رد جی اس دو راه برای دسترسی آسان به المان ها در اختیار برنامه نویس قرار می دهد :

1 - GET : دستور GET برای یافت المان یا المان ها استفاده می شود ؛ این دستور نسبت به دستور SELECT فضای کمتری از حافظه را اشغال می کند .

Get Elements
var elements = Red.get(".heading");

2 - SELECT : دستور SELECT ، ابجکتی را باز میگرداند که به واسطه آن قادر به انجام هرگونه عملیات HTML می باشید .
* تمامی عملیات SELECT در فصل بعد معرفی خواهند شد .

درخواست AJAX

یکی از زیبایی های جاوا اسکریپت ، تکنولوژی AJAX است ؛ این تکنولوژی به زبان ساده ، با ارسال درخواست HTTP به CGI Application و برگرداندن نتیجه به شما در طراحی صفحات پویا کمک زیادی می کند .

از آنجایی که استفاده از آن رایج و بر اساس اصول User Experience ضروری ، رد جی اس تابعی از پیش طراحی شده را به آن اختصاص داده است :

نحوه ارسال درخواست AJAX
var config = {
  type: "POST",
  url: "http://test.com",
  data: { username: "admin" },
  contentType: "application/json",
  dataType: "application/json"
  success: function() {
    var result = this.response;
  }
};

Red.ajax(config);

تابع ajax ، نیازمند ابجکتی حاوی پیکربندی درخواست است ؛

ویژگی type و url که به ترتیب اشاره به نوع و آدرس درخواست داشته ضروری است .

ویژگی data در واقع پارامتر های ارسالی درخواست می باشد .

ویژگی success تابع کال بکی است که در زمان درخواست موفق اجرا می شود .

- المان آیتم -> هر آیتم با خصوصیات CSS این المان نمایش داده خواهد شد

- لیست -> یک آرایه حاوی لیست مربوطه

- پیغام یافت نشد -> در صورتی که نتیجه ای یافت نشد این پیغام نمایش داده خواهد شد

سرویس ارزیابی اعتبار رشته

نحوه کار کردن با سرویس ارزیابی اعتبار رشته :

سرویس ارزیابی اعتبار رشته
var string = 'test';

var result = Red.validate(string, 'required|method:en|limit:0-4');

خاصیت اول به منظور نوع فیلد است :

دو نوع فیلد داریم :

required (نمی تواند خالی باشد)
optional (اختیاری)

خاصیت دوم به منظور متد ارزیابی اعتبار است :

17 نوع متد استاندارد ارزیابی اعتبار داریم :

digit
en
en_digit
fa
fa_digit
mix: فارسی و انگلیسی به همراه اعداد
email: ارزیابی اعتبار ایمیل
username: ارزیابی اعتبار استاندارد نام کاربری (انگلیسی ، اعداد و کاراکتر های خاص .-_)
password: ارزیابی اعتبار استاندارد رمز عبور (انگلیسی ، اعداد و کاراکتر های خاص .@*)
address: آدرس میکس (انگلیسی, فارسی و کاراکتر خاص -)
address_en: آدرس انگلیسی (انگلیسی و کاراکتر خاص -)
address_fa: آدرس فارسی (فارسی و کاراکتر خاص -)
phone
IP: ارزیابی اعتبار آی پی - محدودیت نمی پذیرد
text
text_en
text_fa

خاصیت سوم به منظور محدودیت و تعداد حرف می باشد :

limit:1-10
