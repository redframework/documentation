Overview

Red JS is a Light-Weight Java Script Framework introduced to Provide Easier Relation Between Back and Front Sides ;
Red JS Reached to Version 1.0 and it's Available in Red Framework Enterprise .

Some of Red JS Features :


- Following MVC Pattern

- Coding JS inside HTML Tags

- Access Elements and Modify their Attributes

- AJAX Request

- Cookie Operations

- Graphical Fade

- Validate, Filter, Sanitize Service

Document Ready Event

Document Ready Event is a Pre-defined Event in HTML ; This Event will be Invoked When Document is Fully Downloaded and Rendered .

Assume that We Are going to Register an Event Listener for an Element which it is not Loaded yet ; If We don't use Document Ready Event , An Error Occurs.

There Is a Pre-defined Function in Red JS Which assist you to Use Document Ready Event .

Using Ready Function (Callback)
Red.ready(function () {
    Root.initialize();
});

Module

Module in Red JS refers to Namespace . it comes to allow controllers with equal names .
a document can contain many modules and each modules can contain many controllers .

Register Module
<body>
    <div red-app="main"></div>
    <script src="Dependencies/Library/Red.min.js"></script>
        <script>
            Red.app.module("main");
            Red.ready(function () {
                // some code
            });
        </script>
</body>

Red JS Allows you to Use JavaScript in HTML with Attributes .
in Top Example we choose DIV as Main Module with red-app Attribute and Registered it into Memory with Module Function .

Controller

in Red JS Design Controller Receives Data from Model (Client), Make Decisions By Available Data, Register Data in Memory if Necessary and Refers them to View (Binding).

Registering Controller
<body>
    <div red-app="main">
        <div red-controller="test"></div>
    </div>

    <script src="Dependencies/Library/Red.min.js"></script>
        <script>
            Red.app.module("main");
            Red.app.addController("main", "test", function () {
                // decisions
            });

            Red.ready(function () {
                // some code
            });
        </script>
</body>

In Top Example We Registered Main Module ; then Registered our Controller to Main Module with addController Function .

Third Parameter of addController is a Callback Function . All Variables in That Callback Function will be Registered to Memory for Feature use (Such as Binding) .

You Can Bind your Data through Variables into HTML tags (Data Binding)

Data Binding
<body>
    <div red-app="main">
        <div red-controller="test">
          <div id="result" red-bind="$calculation_result"></div>
        </div>
    </div>

    <script src="Dependencies/Library/Red.min.js"></script>
        <script>
            Red.app.module("main");
            Red.app.addController("main", "test", function () {
                $calculation_result = 3243;
            });

            Red.ready(function () {
                // some code
            });
        </script>
</body>

the Result will Appear in Element with Result ID .


A Controller also Can Start with Initialize Values :

Initialize Values
<body>
    <div red-app="main" red-controller="test" red-init="var user_id = 43;">
        <div id="user-id" red-bind="user_id"></div>
    </div>
    <script src="Dependencies/Library/Red.min.js"></script>
        <script>
            Red.app.module("main");
            Red.app.addController("main", "test", function () {
                // decisions
            });

            Red.ready(function () {
                // some code
            });
        </script>
</body>

In Top Example We Registered test Controller at Main Module ; initialized our Controller with Following Variable : 'user_id'

Then We Bound 'user_id' Variable into 'user-id' element .

Handle Routes

Red JS is Providing a System to Handle Routes and Links in HTML .

Handle Routes via Red JS
<div red-app="main">
    <a href="#!blog">Blog</a>
    <a href="#!forum">Forum</a>
</div>

<script>
    var blog_path = "http://redframework.ir/blog";
    var forum_path = "http://redframework.ir/forums";
    var module_name = "main";
    Red.app.registerRoute(module_name, {
        blog: blog_path,
        forum: forum_path,
    });
</script>

Explain :

We Registered our Routes by registerRoute method in Red.app Module .

registerRoute method Requires 2 Parameter -> Module Name and An Object Containing Route Name as Key and Route Path as Value .

* Routes are Accessible only on their Module's .
* You Can Access Routes in HTML with Special Keys (#!route_name) .

Get / Select Element(s)

Red JS Provides 2 Way to Easy Access Elements .

1 - Get : Get Syntax will do a Query to Find Match Element ; If Only one Element exists will Return Element otherwise it will Return NodeList .

Get Elements
var elements = Red.get(".heading");

2 - Select : Select Syntax will allow You to Do Any HTML Operations on Match Element(s) .
* All Select Operations will be Introduced in Next Chapter .

Select Operations

Select Syntax will Return You a Object which Allows you to Do Any HTML Operation .

Select Operations

// Changing outerHTML of an Element :

// Getting value of an Element :

elements.value("New Value");

// Insert Data After Element :
elements.after("Data");

// Insert Data Before Element :
elements.before("Data");

// Get CSS Property :
elements.CSS("color");

Element Creation

To Have a Dynamic Web Page you Need to Create New Elements , Modify their CSS Properties and Append to an Element as Child Many times .

There is a Pre-Defined Function in Red JS to Help you Do This Faster .

Element Creation
var parent = Red.get("#container");

var css_props = {color: "red"};
var element = Red.create(tag, parent, css_props);

1st Parameter Refers to Tag Name

2nd Parameter Refers to Parent Element (Optional)

3rd Parameter Refers to CSS Properties (Optional)

AJAX Request

One of the Glorious things About Java Script is AJAX Technology ;
This Tech Helps you a lot to Design Dynamic Pages By Sending HTTP Request to CGI Application and Returning Result .

Since Using this Tech is Common and Necessary By User Experience Frames , Red JS Provides a Pre-Defined Function to Do AJAX Request as Fast as Possible :

Sending AJAX Request
var config = {
  type: "POST",
  url: "http://test.com",
  data: { username: "admin" },
  contentType: "application/json",
  dataType: "application/json",
  success: function() {
    var result = this.response;
  }
};

Red.ajax(config);

ajax Function Requires an Object Containing Configuration as Parameter .

type and url Properties are necessary .

data Property is Parameters of Request .

success Property is a CallBack Function Which will be Executed On Successful Request .

Cookie Operations

Cookies are Used to Store Temporary Data in Client Browser .

They will be Stored as String with Pattern :

analytics_token=c956c79e-7209-88ce-74fc-81115ff495ef; red_session_last_activity=11/19/2019

Cookie Management Functions in Red JS are Here to Handle them Faster :

Cookie Management
// Set Cookie :
var cookie_name = "user_logged_in";
var cookie_value = "DSC32KJDEUAPCMJ4";
var expire_days = 7;
Red.setCookie(cookie_name, cookie_value, expire_days);

// Get Cookie :
var result = Red.getCookie(cookie_name);

// Delete Cookie :
Red.deleteCookie(cookie_name);

Custom Timer

Red JS Custom Timer Presents 2 Timer Mode .

00:00 Full Mode (4 Digits)


00 Light Mode (2 Digits)

Implementation :

Custom Timer
// Implementing Full Timer
var full_timer_element = Red.get("#full-timer");
Red.timer(1800, full_timer_element, "full");

// Implementing Light Timer
var light_timer_element = Red.get("#light-timer");
Red.timer(60, light_timer_element, "light");

* Light Mode is Only Suggested for Less than 60 Seconds Periods .

Auto Complete

Auto Complete in Red JS is a Feature Which will be Used on Input Fields and Increases User Exprience .

City :

Implementation :

Necessary Components :


- Input Element -> To Detect Keyup Event and Compare Value to List

- Input Container -> To Append List Element to the Container

- List Element -> Container of List

- Item Element -> Each Item with Dedicated CSS Properties

- List -> An Array Containing the List

- Not Found Message -> A Message which Will be Appear When Value Doesn't Match to List
Auto Complete
var city_input_container = Red.get("#city-input-container");
var city_input = Red.get("#city-input");

var list_element = Red.create("div", null, {
                   "width": "71%",
                   "height": "200px",
                   "background-color": "white",
                   "color": "black",
                   "margin": "auto",
                   "border-radius": "0 0 5px 5px",
                   "padding-top": "4px",
                   "padding-bottom": "4px",
                   "font-family": "'Segoe UI Light', sans-serif",
                   "font-size": "14px",
                   });

var item_element = Red.create("div", null, {
                   "cursor": "pointer",
                   "padding": "2px",
                   "opacity": "0.7",
                   });

var city_list = ["tehran", "babol", "sari",
                "zanjan", "ahwaz", "mashhad", "birjand", "boshehr"];

var not_found_message = "Not Found !";

Red.autoComplete(city_input, city_input_container, list_element, item_element,
                city_list, not_found_message);

* List Element and Item Element Should be Without Any Parent !

* For Element Creation Refer to Element Creation Chapter .

Validation Service

Working With Validation :

VALIDATION SERVICE
var string = 'test';

var result = Red.validate(string, 'required|method:en|limit:0-4');

Attribute 1 No. Of Fields :

there are 2 kind of Fields :

required (which means it can't be null)
optional (which means it can be null)

Attribute 2 Validation Method:

there are 17 Kind of Standard Validation Methods:

digit
en
en_digit
fa
fa_digit
mix: fa and en with digits
email: email Validation
username: standard username validation (English, Digits, Special Chars .-_)
password: standard password validation (English, Digits, Special Chars .@*)
address: address (English, Persian and Special Char -)
address_en: address_en (English and Special Char -)
address_fa: address_fa (Persian and Special Char -)
phone
IP: IP Validation - No Limit is Acceptable for This Method
text
text_en
text_fa

Attribute 3 is Character Limit; For Example:

limit:1-10

Adding Method (Rule) to Validate Service :

ADDING METHOD TO VALIDATE SERVICE
var method = "test";

Red.addValidateRule(method, function (no, min, max, string) {
// DO Your RegEX Based on Info then Return True or False Result
});
