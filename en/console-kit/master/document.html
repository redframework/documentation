Overview

You Can Build Your Advance CLI Applications In PHP with Help of Console Kit .

Configuration

To Get Started with Red Framework Configuration You Need to generate a New Secret Key to Secure your Application.

to Do That You Can Open Red Analytics On Your Project and Click on Project Tab -> Generate New Secret Key

All Path Variables are Available at Config -> Path.php
*** Never Ever Change or Remove Path Variables of Core of Framework
Pathes Should Be Defined (unCommented) By Developer :

DEFINE NECESSARY PATH VARIABLES - EXAMPLE
// MYSQL_PATH - If You Use a MySQL Database in your Project
define('MYSQL_PATH', 'D:\Wamp\bin\mysql\mysql5.7.24\bin\mysql');

// MYSQL_DUMP_PATH - If You Use a MySQL Database in your Project
define('MYSQL_DUMP_PATH', 'D:\Wamp\bin\mysql\mysql5.7.24\bin\mysqldump');

// SQL_CMD_PATH - If You Use a SQL Server Database in your Project
define('SQL_CMD_PATH', '"C:\Program Files\Microsoft SQL Server\Client SDK\ODBC\130\Tools\Binn\SQLCMD.exe"');

// SQLITE_CLI_PATH - If You Use a SQLite Database in your Project
define('SQLITE_CLI_PATH', ROOT_PATH . "database" . DS . "SQLite" . DS . 'sqlite3.exe');

MYSQL_DUMP_PATH & MYSQL_PATH - If You Use a MySQL Database
SQL_CMD_PATH - If You Use a SQL Server Database
SQLITE_CLI_PATH - If You Use a SQLite Database

Backup TimeZone is Based on Your Application TimeZone and Time Format is HH

MIGRATION :

If Your Turn Your Database Migration On, Framework Will Try to Migrate Your Current Database (Stored at database -> Migration -> migration_of_db1.sql) to Your Connected Database.

Migrate Process Will be Done By Shell ; So Before that You Have to Set These 4 Path in Path Variables .

MYSQL_DUMP_PATH & MYSQL_PATH - If You Use a MySQL Database
SQL_CMD_PATH - If You Use a SQL Server Database
SQLITE_CLI_PATH - If You Use a SQLite Database

Working with ORM

Red Framework ORM Provides 4 Base Operations CRUD (Create, Read, Update, Delete) and 2 Extra Operations (COUNT, AVG)


All Database Engine Flags (Const. For Easier Usage) :

DATABASE ENGINE FLAGS
/* DB1 - Which Refers to Database Slot 1
DB2 - Which Refers to Database Slot 2
DB3 - Which Refers to Database Slot 3
CREATE - Which Refers to Create Operation
READ - Which Refers to Read Operation
READ_CONDITION - Which Refers to Read Operation By Condition
UPDATE - Which Refers to Update Operation
UPDATE_CONDITION - Which Refers to Update Operation By Condition
DELETE - Which Refers to Delete Operation
DELETE_CONDITION - Which Refers to Delete Operation By Condition
COUNT - Which Refers to Count Operation
COUNT_CONDITION - Which Refers to Count Operation By Condition
AVG - Which Refers to Average Operation
AVG_CONDITION - Which Refers to Average Operation By Condition
FREE - Which Refers to Free Query */



Note : All ORM Operations Should be Done in A Model

Create Operation :

ORM - CREATE
$this->setFields([
'page' => 'any',
'hits' => 'any',
]);

$this->setParameters([
'page' => 'singup',
'hits' => '0',
]);

$this->query("hits", CREATE, DB1);


Explain :

At The First Command We Called 'SetFields' Method of Model Class With 1 Parameter (@array) Which Contains Key as Field Name and Value as Model Validation Roles ; We Have Set 'any' for Roles Which Means Anything Would Be Accepted and No Validation is Needed.


At The Second Command We Called 'SetParameters' Method of Model Class With 1 Parameter (@array) Which Contains Key as Field Name and Value as Field Value ;


At Third and Final Command We Called Query Method of Model Class With 3 Parameters (Parameters in ltr Sort : Tables, Operation Flag, Database Name Flag)

Read Operation :

ORM - READ
$this->setFields([
'page' => 'any',
'hits' => 'any',
]);

$this->query("hits", READ, DB1, "id DESC", $offset, $limit);


Explain :

We Have Set Page and Hits Fields to Fetch Data from with SetFields Method .


At Final Command We Call Query Method with 6 Parameters (in ltr Sort : tables, CRUD Flag, Database Slot Flag, order, offset, limit).

Order By Descending : FieldName DESC
Order By Ascending : FieldName ASC
Last 3 Parameters are Optional.

Read Operation By Condition :

In Following Example We Fetch Data By Condition in an Easier Way with Design Pattern :

ORM - READ_CONDITION
$this->setFields(['id' => 'any', 'page' => 'any', 'hits' => 'any', ])->
setConditionFields(['page' => 'any|equal|operator:and', 'hits' => 'any|equal'])->
setConditionParams(['page' => 'singup', 'hits' => '60'])->
query("hits", READ_CONDITION, DB1, "id DESC");


Explain :

We Have Requested 3 Fields with setFields Method .


After that We have Set Our Condition Fields and Attributes with setConditionFields
1st token 'Any' Means no Validation is Needed, 2nd token Named equal is Conditional Operator (greater, smaller, equal, like, not_like)
3rd token is Relational Operator (and, or, ...)

There is No Relational Operator is Needed for Last Condition


Note : Attributes and Model Validation Will Be Introduced Totally at the End of this Chapter

after Setting Conditions it's time to set Condition Parameters with setConditionParams method

and finally we Call Query method which execute our query hits table by id descending order.


After Querying you Can Fetch Value easily with Following Syntax :

ORM - READ_CONDITION (FETCH DATA)
$result = $this->getContent(DB1);

Update Operation :

In Following Example We Update Record in Database :

ORM - UPDATE
$this->setFields(['email' => 'any'])->
setParameters(['email' => 'mahbod@gmail.com'])->
query("users", UPDATE, DB1);


Explain :

At Above Example We Updated any Email Field to 'mahbod@gmail.com' .

Update Operation By Condition :

In Following Example We Update Record in Database By Condition and Validation :

ORM - UPDATE_CONDITION
$this->setFields(['email' => 'any'])->
setParameters(['email' => 'mahbod@gmail.com'])->
setConditionFields(['username' => 'required|method:en_digit|limit:5-25|sanitize_space:yes|equal',])->
setConditionParams(['username' => 'mahbod'])->
query("users", UPDATE_CONDITION, DB1);


Explain :

At Above Example We Updated Email Field to 'mahbod@gmail.com' where username is 'mahbod' .

Model Validation Note : we Validate username field with Following Attributes ; english and digits format, between 5-25 letters, sanitize extra space to match case

Model Validation will be Totally introduced at the End of this Chapter

Delete Operation :

In Following Example We Delete Records in Database :

ORM - DELETE
$this->query('hits', DELETE, DB1);

Delete Operation By Condition :

In Following Example We Delete Records in Database By Condition :

ORM - DELETE_CONDITION
$this->setFields(['hits'=> 'any']);
$this->setConditionFields(['hits' => 'any|equal']);
$this->setConditionParams(['hits'=>'mahi']);

$this->query('hits', DELETE, DB1);

Count Operation :

ORM - COUNT
$this->query('hits', COUNT, DB1);

$result = $this->getContent(DB1);

Count Operation By Condition :

ORM - COUNT_CONDITION
$this->setConditionFields(['id' => 'any|equal']);
$this->setConditionParams(['id' => '12']);
$this->query('test', COUNT_CONDITION, DB2);
$result = $this->getContent(DB2);

Average Operation :

ORM - AVG
$this->setFields(['rate' => 'any', ]);
$this->query('post_votes', AVG, DB1);

Average Operation By Condition :

ORM - AVG_CONDITION
$this->setFields(['rate' => 'any', ]);
$this->setConditionFields(['post_id' => 'any|equal|operator:and',]);
$this->setConditionParams(['post_id' => $id['id'],]);

$this->query('post_votes', AVG_CONDITION, DB1);

Free Query :

ORM - Free Query
$this->freeQuery("UPDATE `users` SET `email` = 'mzero651@gmail.com' WHERE `username` = 'mahbod'", READ_CONDITION, DB1);

$result = $this->getContent(DB1);


Explain :

At Above Example We did a SQL Query (Read Condition Method) with Database 1 Slot .

Model Validation :

in this topic We are Going to Explain All Model Validation Rules and techniques.

ORM - MODEL VALIDATION
$this->setFields(['username' => 'required|method:en_digit|limit:5-25|sanitize_space:yes']);

Attribute 1 No. Of Fields :

there are 3 kind of Fields :

required (which means it can't be null)
optional (which means it can be null)
any (which means no validation is needed)

Attribute 2 Validation Method:

there are 17 Kind of Standard Validation Methods:

digit
en
en_digit
fa
fa_digit
mix: fa and en with digits
email: email Validation
username: standard username validation (English, Digits, Special Chars .-_)
password: standard password validation (English, Digits, Special Chars .@*)
address: address (English, Persian and Special Char -)
address_en: address_en (English and Special Char -)
address_fa: address_fa (Persian and Special Char -)
phone
IP: IP Validation - No Limit is Acceptable for This Method
text
text_en
text_fa

Attribute 3 is Character Limit; For Example:

limit:1-10

Attribute 4 is Sanitizing Extra Spaces;

value can be yes / no.

Pagination

in Red Framework Pagination will be Calculated right after you executed a READ or READ_CONDITION Query with Offset and Limit

For Example:

PAGINATION
$this->setFields([
'page' => 'any',
'hits' => 'any',
]);

$this->query("hits", READ, DB1, "id DESC", 0, 10);

$pages = $this->getPagesCount(DB1);

Error Handling

Error Handling in Red Framework :

Pushing a Non-Fatal Error (Which Will Stop ORM and etc From Processing) :

Non-Fatal Error

Red::pushError($error);

Fatal Error
use Red\Red;

Red::generateError($error_no, $error_message);
